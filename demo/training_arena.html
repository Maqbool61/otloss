Why Wasserstein loss?

Cross-entropy and MSE have a fundamental flaw: their gradients vanish when the model and target distributions don't overlap. Wasserstein-2 defines a geometric distance using the ground metric of the output space — it always produces meaningful gradients.

Wasserstein-p distance: Wₚ(μ,ν) = inf_γ ∫ ‖x-y‖ᵖ dγ(x,y)

Computed via entropy-regularised Sinkhorn iterations in log-domain — O(n²) per step vs O(n³) for exact OT.

Choosing blur (ε = blur²)

blur = 0.5
Fast / biased

Rule of thumb: blur ≈ std(data) × 0.05. Sinkhorn divergence debiasing (debias=True) corrects entropic bias at any blur level.

Quick start — real code

Install:

pip install otloss

Drop-in replacement for MSELoss:

from otloss import WassersteinLoss

criterion = WassersteinLoss(p=2, blur=0.05)
# pred / target shape: (batch, N_particles, D_dims)
loss = criterion(pred, target)
loss.backward()

For large N or high D — use the O(n log n) variant:

from otloss import SlicedWassersteinLoss

criterion = SlicedWassersteinLoss(n_projections=200)
loss = criterion(generated, reference)
loss.backward()
